# 產品需求文件 (PRD) - 任務管理系統

## 1. 專案概述

### 背景與動機
在快節奏的現代生活中，個人往往因為繁雜的日常事務而感到焦慮且容易遺忘重要事項。為了幫助個人更有效地追蹤待辦事項、提升工作與生活效率，我們需要一個簡單、直覺的任務管理系統。

### 目標用戶
需要管理日常工作的個人用戶。

### 核心價值主張
提供一個輕量、無負擔的待辦事項管理工具，專注於任務的紀錄與狀態追蹤，協助使用者有效管理時間並專注於執行。

## 2. 功能需求

以下列出系統的主要功能需求與對應的使用者故事：

1. **新增任務 (Create Task)**
   - 作為「個人用戶」，我希望「可以輸入任務名稱並新增任務」，以便「記錄需要完成的事情」。
2. **查看任務列表 (View Task List)**
   - 作為「個人用戶」，我希望「可以一次看到所有待辦和已完成的任務清單」，以便「掌握整體的工作進度」。
3. **標記完成 (Mark Task as Complete)**
   - 作為「個人用戶」，我希望「可以將特定任務標記為『已完成』」，以便「清楚區分哪些事情已經處理完畢」。
4. **編輯任務內容 (Edit Task)**
   - 作為「個人用戶」，我希望「可以修改現有的任務名稱或內容」，以便「在任務細節改變或輸入錯誤時進行更正」。
5. **刪除任務 (Delete Task)**
   - 作為「個人用戶」，我希望「可以刪除不再需要的任務」，以便「保持任務清單乾淨整潔」。

## 3. 非功能需求

### 技術限制
- **後端框架**：Flask (Python)
- **前端模板**：Jinja2 (Server-Side Rendering)
- **資料庫**：SQLite (輕量級關聯式資料庫)

### 效能與安全考量
- **效能考量**：操作反應時間應順暢無延遲，頁面載入速度需符合基本標準。
- **安全防護**：
  - 任務名稱及內容等輸入欄位需防範 XSS 攻擊 (跨站腳本攻擊)，並由 Jinja2 預設轉義處理。
  - 防範基本的 CSRF 攻擊，確保表單提交來源正確。
- **資料保護**：系統需能正確且穩定地寫入 SQLite，重開系統時資料不可遺失。

## 4. MVP 範圍 (Minimum Viable Product)

- **Must Have (必須包含)**：
  - 新增任務功能
  - 查看全部任務列表
  - 將任務標記為完成/未完成
  - 刪除任務功能
- **Should Have (應該包含)**：
  - 編輯任務內容功能
- **Nice to Have (有會更好，可留待未來擴充)**：
  - 任務到期日設定
  - 任務分類（如工作、個人、購物清單）
  - 任務拖曳排序
